**Helper methods:**

**`_prepare_context()`** — Build Jinja2 template context
- Project metadata: name, slug, description, timestamp
- Opportunity data: pain_points, total_score
- Analysis data: tech_stack, dependencies, complexity, hours, features, risks
- Database config: db_user, db_password, db_name
- Models: extracted from features (see below)

**`_extract_models()`** — Database model extraction heuristic
- Input: MVP features (list of strings)
- Output: List of model dicts: `[{"name": "Payment", "table_name": "payments", "fields": [...]}, ...]`
- Algorithm:
  1. Scan features for keywords: `user`, `dashboard`, `payment`, `subscription`, `project`, `task`
  2. Map keywords to predefined model templates
  3. Each template includes: name, table_name, fields (name, type, python_type, optional, nullable, default)
- Default: At least one model (`Project`) if no keywords found
- Common models:
  - **Profile:** bio, avatar_url
  - **Dashboard:** name, config
  - **Payment:** amount, currency, status
  - **Subscription:** plan_name, status, expires_at
  - **Project:** name, description, status
  - **Task:** title, description, completed

**`_slugify()`** — Convert project name to slug
- Lowercase
- Replace non-alphanumeric with `-`
- Strip leading/trailing `-`
- Example: "AI-Powered Code Review Tool" → "ai-powered-code-review-tool"

## Model Extraction Heuristic

**Input:** MVP features (list of strings)  
**Example:** `["user_dashboard", "payment_processing", "project_management"]`

**Algorithm:**
1. Scan each feature for keywords: `user`, `dashboard`, `payment`, `subscription`, `project`, `task`
2. Map keyword → predefined model template
3. Collect unique models (set)
4. Default: If no keywords found, add `Project` model

**Predefined models:**

| Keyword | Model Name | Table Name | Fields |
|---------|------------|------------|--------|
| `user` | Profile | profiles | bio, avatar_url |
| `dashboard` | Dashboard | dashboards | name, config |
| `payment` | Payment | payments | amount, currency, status |
| `subscription` | Subscription | subscriptions | plan_name, status, expires_at |
| `project` | Project | projects | name, description, status |
| `task` | Task | tasks | title, description, completed |

**Field structure:**
```python
{
    "name": "amount",
    "type": "Integer",  # SQLAlchemy type
    "python_type": "int",  # Pydantic type
    "optional": False,  # Pydantic optional (|  None)
    "nullable": False,  # SQLAlchemy nullable
    "default": None,  # Default value
}
```

**Example output:**
```python
[
    {
        "name": "Dashboard",
        "table_name": "dashboards",
        "fields": [
            {"name": "name", "type": "String", "python_type": "str"},
            {"name": "config", "type": "String", "python_type": "str", "optional": True, "nullable": True},
        ]
    },
    {
        "name": "Payment",
        "table_name": "payments",
        "fields": [
            {"name": "amount", "type": "Integer", "python_type": "int"},
```

## Safety Gate (Manual Approval)

**Purpose:** Prevent automatic MVP generation for risky or low-confidence analyses.

**Workflow:**
1. Analysis completes → `approved = FALSE` (default)
2. Human reviews analysis: `GET /api/v3/business/opportunities/{id}/analysis`
3. If satisfactory → Manual approval: `UPDATE opportunity_analyses SET approved = TRUE WHERE id = X`
4. MVP generator checks: `WHERE approved = TRUE` before generating code

**Future enhancement (P3.4):**
```python
PATCH /api/v3/business/opportunities/{id}/analysis
Body: {"approved": true, "approved_by": "maxence"}
```

**Safety check in API:**
```python
if not analysis.approved:
    raise HTTPException(
        status_code=400,
        detail="Analysis is not approved. Set approved=true before generating MVP."
    )
```
