# Cybersecurity Skills — Evaluation Criteria

## Quality Dimensions

### 1. Technical Accuracy (40%)
- Correct use of security terminology and frameworks
- Accurate tool references and capabilities
- Valid CVE/CWE references when applicable
- Proper risk classification (CVSS, severity levels)

### 2. Completeness (25%)
- All required sections present in output
- Evidence-based findings with supporting data
- Clear remediation steps with prioritization
- Timeline and resource estimates

### 3. Actionability (20%)
- Specific, implementable recommendations
- Prioritized by risk level and effort
- Clear ownership and responsibility assignment
- Measurable success criteria

### 4. Compliance Alignment (15%)
- References to relevant standards (NIS2, GDPR, ISO 27001)
- Mapping to regulatory requirements
- Audit-ready documentation format
- Evidence chain integrity

## Domain-Specific Criteria

### Blue Team
- Detection rule effectiveness (false positive rate)
- Coverage of MITRE ATT&CK techniques
- Integration with existing SIEM/EDR stack
- Incident response time improvement potential

### Red Team
- Exploitation chain validity
- Business impact assessment accuracy
- Remediation priority correctness
- Safe execution verification (no production impact)

### Compliance
- Regulatory coverage completeness
- Gap identification accuracy
- Implementation timeline realism
- Cost estimation accuracy

### OSINT
- Source reliability assessment
- Data freshness verification
- Attribution confidence levels
- Legal boundary compliance
