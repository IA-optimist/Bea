### **3. Feasibility Analyzer**
**File:** `core/business/feasibility_analyzer.py` (352 LOC)

**Class:** `FeasibilityAnalyzer`

**Method:** `async def analyze(opportunity: Opportunity, project_id: int = 1) -> Dict[str, Any]`

**Process:**
1. Build analysis prompt (12-section structured format)
2. Create mission dict: `{"mission_id", "goal", "project_id", "metadata", "user_input"}`
3. Execute cognition: `await cognition.execute_mission_with_cognition(mission, enable_tot=True, enable_confidence=True, enable_learning=True)`
4. Parse result (regex + JSON extraction)
5. Return structured analysis

**Cognition configuration:**
- Enable Tree-of-Thought: `enable_tot=True`
- Enable self-confidence scoring: `enable_confidence=True`
- Enable active learning: `enable_learning=True`
- Duration: 4-8 minutes (cognition overhead)

**Prompt structure (12 sections):**
1. RECOMMENDATION: BUILD | SKIP | NEEDS_MORE_RESEARCH
2. REASONING: 2-3 sentence explanation
3. TECH_STACK: ["python", "fastapi", "react", "postgresql"]
4. DEPENDENCIES: ["stripe", "sendgrid", "openai"]
5. COMPLEXITY_SCORE: 1-10 (1=trivial, 10=extremely complex)
6. ESTIMATED_HOURS: Integer
7. MVP_FEATURES: ["user_auth", "dashboard", "api", "payments"]
8. NICE_TO_HAVE: ["social_login", "advanced_analytics"]
9. OUT_OF_SCOPE: ["white_label", "enterprise_sso"]
10. TECHNICAL_RISKS: ["API rate limits", "Complex auth flow"]
11. MITIGATION: ["Implement caching", "Use OAuth library"]
12. MARKET_FIT_SCORE: 0-100

**Parser:**
- Regex patterns for single-line fields (RECOMMENDATION, COMPLEXITY_SCORE, ESTIMATED_HOURS, MARKET_FIT_SCORE)
- Multi-line text extraction (REASONING)
- JSON array parsing for lists (tech_stack, dependencies, features, risks)
- Fallback: Comma-split on JSON parse failure
- Safe defaults on parse errors

Trigger feasibility analysis (background task).

**Query params:**

**Process:**

5. Update `opportunity.analyzed = TRUE`

  }
}
```

**Background task:**
- Async function `_run_analysis()`
- Runs `await analyzer.analyze()`
- Stores in DB, commits transaction
- Logs: start, completion, errors

## Manual Approval Gate

**Safety feature:** `approved` column in `opportunity_analyses` table

**Purpose:** Prevent automatic MVP generation for low-confidence or risky analyses

**Workflow:**
1. Analysis completes → `approved = FALSE` (default)
2. Human reviews analysis (GET `/api/v3/business/opportunities/{id}/analysis`)
3. If satisfactory → Manual approval: `UPDATE opportunity_analyses SET approved = TRUE WHERE id = X`
4. P3.3 MVP generator checks: `WHERE approved = TRUE` before generating code

**Future API endpoint (P3.3):**
```

Body: {"approved": true, "approved_by": "maxence"}
